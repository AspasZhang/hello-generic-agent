# Case 6：内审报告人机协作写作

> **学完本章，你将掌握如何让 GA 与审计专家高效协作——口述十分钟出提纲、分批给素材出初稿、逐轮审阅出终稿，彻底告别格式地狱**

> **📍 For GA · 给智能体看的输入**
> 
> ```
> 我要写一份内审报告。我们按三阶段协作来：
> 
> 阶段1（口述→提纲）：我先口述审计背景、目的和初步发现，你听完不要直接写报告——先确认听懂，追问逻辑缺口直到闭合，然后出提纲草案。我没有说「提纲没问题」之前，你不要进入下一阶段。
> 
> 阶段2（素材→初稿）：提纲确认后，你列素材清单告诉我需要哪些数据，我分批给你Excel/PDF/照片。你提取数据汇成表格，先出Markdown纯文本初稿——先别急着套docx格式，内容确认前格式会分散注意力。
> 
> 阶段3（审阅→定稿）：我逐轮审阅初稿，你按我的意见修改。确认无误后输出docx并归档协作日志。
> 
> 底线：你不做定性判断、不替我做人员追责。凡是涉及「某某人存在XX问题」「建议追责XX」的措辞，一律留[请确认：…]标记。
> ```
> 
> 上面这段是用户原话，**直接复制粘贴给 GA 就能跑**——读者无需关心 GA 怎么拆解。 下面是给**人类读者**看的章节解析：GA 把这一句话拆成了哪些步、为什么这么拆、踩了哪些坑。

---

## 🎯 学习目标

1. 理解「口述→提纲→素材→初稿→定稿」的三阶段流水线，以及每个阶段的「当前你可以」选项机制
2. 学会让 GA 从 Excel/PDF/照片中提取数据并汇总为报告表格
3. 掌握协作日志恢复协议——跨会话、跨模型实例都能无缝继续
4. 理解人机分工红线：GA 做执行和整理，人做判断和决策

---

## 19.1 适用场景

> 💡 内审报告通常有固定的格式要求、需要引用大量数据表、经历多轮审阅修改。传统方式下，审计专家口述思路后需要手动排版、反复调格式、来回改措辞。GA 可以把这条链路自动化——你负责判断，GA 负责执行。

本方法的核心思想：**文本先行**。初稿先出 Markdown 纯文本审内容，确认无误后再套 docx 格式。理由很简单——如果一上来就是排版精美的 Word，你的注意力会被字体、表格边框、页码分散，反而忽略内容问题。

---

## 19.2 一个具体场景

为便于理解，本章用一个简化但真实的内审场景贯穿全程：

> 审计部在抽查中发现，某物业公司管理的商场广告位租金收入和实际合同金额不一致——有几笔广告位合同已签但钱没入账。需要写一份**专项审计报告**，查清问题、给出处理建议。

下面我们用这个场景，一步步演示 GA 和审计专家的协作全过程。

---

## 19.3 整体流程概览

```
┌─────────────────┐
│ 阶段1：口述→提纲  │   专家口述背景/发现 → GA 追问清单+提纲草案
│ （多轮直到闭合）  │   终止条件：专家说「提纲没问题」
└────────┬────────┘
         ▼
┌─────────────────┐
│ 阶段2：素材→初稿  │   GA 列素材清单 → 专家分批给文件 → GA 提取数据
│ （文本先行）      │   确认后 GA 输出 Markdown 初稿
└────────┬────────┘
         ▼
┌─────────────────┐
│ 阶段3：审阅→定稿  │   专家逐轮审阅 → GA 修改 → 格式预检 → 输出 docx
│ （终稿归档）      │   同时归档协作日志，方便日后恢复
└─────────────────┘
```

> ⚠️ **禁止跳过确认自动推进**。每个阶段末尾，GA 必须列出「当前你可以」选项，由审计专家拍板。

---

## 19.4 人机分工红线

在开始之前，必须明确一件事：**GA 不做定性判断、不做人员追责**。以下是铁律：

| 审计专家 | GA（星期五） |
|---------|------------|
| 事实是否准确 | 逻辑是否闭合 |
| 定性是否恰当 | 表达是否规范 |
| 提供数据和证据 | 套格式、排表格、统编号 |
| 拍板建议措辞 | 追问遗漏与矛盾 |

> ❌ **红线**：凡是涉及「某某人存在XX问题」「建议追责XX」的措辞，GA 一律留 `[请确认：...]` 标记，绝不代笔。

---

## 19.5 阶段1：口述 → 提纲

### 19.5.1 审计专家启动

审计专家只需口述（打字、语音转文字、列要点都可以）三件事：

- **为什么做**：这次审计的背景
- **想查清什么**：审计目的
- **发现了什么**：查了哪些科目、流程、人员，具体发现了什么问题

以我们的物业公司场景为例，愚者先生说：

```
我简单说下情况。我们对XX物业公司2025年度进行了专项审计，重点查商场广告位的租金收入。
发现三个问题：一是2025年3-5月期间，商场东侧入口LED屏的租金收入没有入账，
合同显示月租3万元，但财务账上找不到这笔钱；
二是7-8月商场中庭广告位的实际租用方跟合同签约方不一致，
疑似存在转租但未报备；
三是全年广告位合同管理混乱，合同编号不连续，
有3份合同只有复印件没有原件。
我们需要写一份专项审计报告，把问题查清楚，给物业公司下整改通知。
```

### 19.5.2 GA 第一轮回应

GA 听完口述后，不能直接写报告——必须先做三件事：**确认听懂、追问缺口、出提纲草案**。

GA 的回应模板如下（以实际对话为例）：

```
✅ 理解摘要：
对XX物业公司2025年度商场广告位租金收入进行专项审计，发现：
① LED屏3-5月租金未入账（合同月租3万→共9万）
② 中庭广告位存在疑似转租未报备
③ 合同管理混乱（编号不连续、3份仅有复印件）
需出具专项审计报告，要求整改。

❓ 追问清单：
1. 审计的时间范围是2025全年还是特定月份？
2. 共涉及几个广告位？LED屏+中庭，还有其他的吗？
3. 「未入账」的钱最终去了哪里——是打到了公司账户但财务没记，
   还是根本没打到公司账户？
4. 转租问题——实际租用方是谁？有没有转租协议？
5. 合同管理混乱——一共多少份合同？编号不连续的范围是？
6. 除了这三个问题，还查过物业费、停车费等其他收入吗？

⚠️ 逻辑断裂点：
- 「钱没入账」缺少关键环节：是没打款、打了没记、还是打了记错科目？
- 「疑似转租」缺少证据：实际使用方是谁、交租给谁？

📝 提纲草案
主题：专项审计报告
★ 主线：广告位租金收入专项审计
· 辅线：合同管理规范性

结构框架：
  一、审计概况（背景、目的、范围、方法）
  二、审计发现
      2.1 LED屏租金未入账
      2.2 中庭广告位转租未报备
      2.3 合同管理混乱
  三、原因分析
  四、处理建议
  五、整改要求

预期篇幅：约8000字，含3个数据表
```

### 19.5.3 多轮追问直到闭合

口述很少一次就逻辑完整。GA 的核心价值之一就是**追问到闭合**——不放过任何逻辑缺口。以下是追问的典型触发条件：

| 触发条件 | GA 的行为 |
|---------|-----------|
| 口述中出现未解释的金额/人名/时间 | 追问到闭合 |
| 两个说法前后矛盾 | 指出矛盾，请专家澄清 |
| 专家回答后引出新疑点 | 继续追问，不限轮次 |
| 专家说「漏了什么」 | 按报告结构逐节排查 |

**终止条件只有一个**：审计专家说「提纲没问题 / 可以了」。

### 19.5.4 提纲确认

阶段1结束时，GA 必须列出选项，不能自动推进：

```
📋 当前你可以：
  → A：「提纲OK，进入阶段2列素材清单」
  → B：「第X点需要调整……」
  → C：「我去查一下，稍后回复」
```

## 19.6 阶段2：素材 → 初稿

阶段2的核心理念：**先别写报告，先列清单**。搞清楚需要什么数据，再让专家提供。

### 19.6.1 GA 列出素材清单

进入阶段2后，GA 根据提纲自动生成素材清单。以物业公司场景为例：

```
📋 素材清单：

□ 数据表：「2025年1-12月广告位租金收入明细表」
  → 需含：广告位名称、合同金额、实收金额、收款日期、差额

□ 对比表：「应收费 vs 实收费对比表」
  → 按广告位分列，标出差异

□ 合同文件：「LED屏租赁合同」「中庭广告位合同」
  → 确认租期、金额、付款方式条款

□ 银行流水：「2025年3-5月公司账户银行流水」
  → 确认是否有对应3笔3万元的入账记录

□ 现场照片：「商场东侧LED屏实物照片」「中庭广告位实物照片」
  → 证明广告位确实存在且正在使用

□ 制度文件：「物业公司合同管理制度」
  → 确认合同编号、存档的规范要求

□ 其他：OA审批记录（广告位合同审批流程截图）
```

### 19.6.2 分批处理素材

专家不会一次性给齐所有素材——这是正常的。流程是：

```
清单 → 给一批文件 → GA 提取摘要 → 确认 → 给下一批 → … → 发令「生成初稿」
```

专家给文件时，GA 根据文件类型选择不同工具处理：

| 文件类型 | 工具 | 处理方式 |
|---------|------|---------|
| **Excel (.xlsx)** | python-docx + openpyxl | 提取关键列（科目/金额/日期/部门），汇总为报告表格 |
| **Word (.docx)** | python-docx | 提取段落文字，定位关键条款 |
| **PDF 扫描件** | mmx vision describe | OCR 提取合同条款、凭证编号、金额 |
| **照片** | mmx vision describe | 描述现场 + OCR 提取可见文字 |
| **PowerPoint** | python-pptx | 提取幻灯片文字内容 |

> ⚠️ 禁止使用 LibreOffice ODF 格式（.ods/.odt），输入输出均为 Microsoft 格式。

### 19.6.3 实际素材处理示例

假设专家发了「广告位租金明细表.xlsx」，GA 处理过程如下：

```
📊 已提取：「广告位租金明细表.xlsx」

关键发现：
- 共7个广告位，全年合同金额合计 126万元
- 实收 117万元，差额 9万元
- 差额集中在 LED屏（3-5月，每月3万，共9万未收）

数据摘要：
| 广告位 | 合同金额 | 实收 | 差额 | 状态 |
|--------|---------|------|------|------|
| LED屏 | 36万 | 27万 | 9万 | 3-5月未收 |
| 中庭 | 24万 | 24万 | 0 | 正常 |
| ... | ... | ... | ... | ... |

📋 当前你可以：
  → A：「数据解读无误，继续」
  → B：「XX地方理解不对……」
  → C：「素材够了，生成初稿」
```

> 💡 缺失项不要阻塞流程——在初稿中标注 `[需补充：XX数据]` 即可。

### 19.6.4 生成初稿（文本先行）

所有素材确认后，GA 以 Markdown 格式先生成初稿。**为什么先出文本而不是直接出 Word？** 因为文本审阅快速直接——80% 的内容修改可以在文本阶段完成，避免被格式分散注意力。

初稿需要附带的关键标记：

- `[请确认：XX定性是否准确？]` —— 涉及人员定性或追责措辞
- `[需补充：XX数据]` —— 缺失的素材
- `[推测：XX]` —— 从上下文推断但不确定的内容

交付初稿时：
```
📋 当前你可以：
  → A：「初稿可以，开始审阅」
  → B：「XX部分需要修改……」
  → C：「缺了XX数据，补充后再出初稿」
```

## 19.7 阶段3：审阅 → 定稿

### 19.7.1 逐轮审阅

阶段3的工作方式与阶段1类似——多轮来回。专家审阅初稿后可能说：

- 「第X点改成……」 → GA 直接修改对应位置
- 「这里缺XX数据」 → GA 标记 `[需补充]`，继续其他部分
- 「可以终稿」 → GA 执行格式预检

每轮修改后附带选项：
```
📋 当前你可以：
  → A：「第X点确认无误」
  → B：「第X点改成……」
  → C：「这里缺XX数据」
  → D：「可以终稿，输出文件」
```

### 19.7.2 写作风格铁律

GA 在生成文本时必须遵守的铁律：

| # | 铁律 | 示例 |
|---|------|------|
| 1 | 去AI味：不用华丽辞藻 | ❌「极大地提升了管理效能」 → ✅「比上年度减少了3次漏记」 |
| 2 | 结论必讲依据 | ❌「合同管理不规范」 → ✅「3份合同无编号、2份无原件」 |
| 3 | 禁止大话空话 | ❌「进一步加强管理」 → ✅「要求2026年6月前补全合同编号」 |
| 4 | 表格后有讲解 | 每个数据表下面必须有一段文字解读 |
| 5 | 证据链完整 | 事实 → 数据 → 分析 → 结论，每步有支撑 |

> 🔒 以上铁律优先级高于 GA 从历史报告中学习的任何风格。

### 19.7.3 格式预检

专家说「可以终稿」后，GA 不能直接输出 docx——先跑一遍预检：

```
🔍 格式预检结果：
□ ✅ 全文约 8200 字（7000–10000 范围）
□ ✅ 无空话词（「进一步加强」「持续优化提升」等未检出）
□ ✅ 每条建议含「谁+做什么+时限」
□ ✅ [请确认] 标记已全部清除
□ ✅ 3个表格各有讲解段落
□ ✅ 证据链闭合
```

预检通过后，GA 用 python-docx 生成 .docx 文件，套用标准内审报告格式。

> ⚠️ **逃生舱**：如果 python-docx 生成脚本连续 3 次失败，放弃自动化方案，改用手工方式——打开已生成的 .docx，逐段修改段落文本。手工改比反复调脚本更可靠。

## 19.8 协作日志：跨会话恢复的秘密

内审报告通常不是一次会话能写完的。下次打开 GA 时，它怎么知道自己上次做到哪了？

答案是**协作日志**——GA 在阶段1启动时自动创建，每次操作后自动更新。

### 19.8.1 日志包含什么

```
collab_log_物业公司广告费_20260511.md

## 一、对话索引      ← model_responses 文件+行号，方便回看原对话
## 二、项目概要      ← 报告类型、主线辅线、预期篇幅
## 三、当前提纲      ← 实时更新的最新版本
## 四、追问追踪表    ← 每条追问的状态（待答/已答）+ 答复摘要
## 五、素材清单      ← 文件路径 + 提取摘要 + 处理状态
## 六、关键决策记录  ← 每次 A/B/C/D 选择
## 七、时间线        ← 阶段切换节点
```

### 19.8.2 追问追踪表示例

```
| # | 追问内容 | 状态 | 答复摘要 |
|---|---------|------|---------|
| 1 | 审计时间范围是2025全年？ | ✅已答 | 是的，2025.01-2025.12 |
| 2 | 未入账的钱最终去哪了？ | ✅已答 | 根本没打到公司账户，物业经理私收 |
| 3 | 转租的租用方是谁？ | ⏳待答 | — |
| 4 | 一共多少份广告位合同？ | ✅已答 | 15份，其中3份无原件 |
| 5 | 还查过其他收入吗？ | ✅已答 | 只查了广告位，未查其他 |
```

### 19.8.3 日志写入节点

GA 在以下时刻自动写入日志（不需要专家提醒）：

| 时机 | 写入内容 |
|------|---------|
| 阶段1第一轮回应后 | 项目概要 + 追问追踪表 + 当前提纲 |
| 每次专家回答问题 | 更新对应追问的状态+答复 |
| 每次 A/B/C/D 选择 | 追加关键决策记录 |
| 阶段2收到素材 | 追加素材清单 |
| 阶段3每轮修改 | 追加修改记录 |

每次写入后 GA 主动报告：`📝 日志已更新：[具体追加了哪些内容]`。

### 19.8.4 会话恢复协议

新会话打开时，GA 按以下顺序恢复状态：

```
1. 打开协作日志
2. 先读「追问追踪表」——避免重复提问
3. 再读「当前提纲」和「素材清单」——知道做到哪了
4. 最后读「关键决策记录」和「时间线」——确认当前阶段
5. 向专家确认：「上次我们做到了XX阶段，追问还差3条未回复，继续吗？」
```

> ❌ **禁止行为**：不读追问追踪表就重新问已经回答过的问题。禁止跨 model_responses 全文搜索猜测关键词。

## 19.9 常见坑与规避

| 坑 | 表现 | 规避方法 |
|----|------|---------|
| 口述没交代审计范围 | 提纲漏掉整块内容 | 阶段1首轮追问必含「审了哪些时间段/科目/部门」 |
| 主要矛盾被次要淹没 | 9万的未入账和3份合同混在一起 | 提纲阶段明确标「★主线」与「·辅线」 |
| 建议笼统化 | 「建议加强合同管理」 | 初稿检查每条建议是否有「谁+做什么+时限」 |
| 危险措辞越界 | 「XX存在贪污嫌疑」 | 留 `[请确认]`，宁多勿漏 |
| 格式崩坏 | 表格跨页断裂、编号错乱 | 终稿前跑格式预检 |

---

## 19.10 一句话总结

人机协作写报告的核心不是「AI 替你写」，而是**AI 做执行和整理，你做判断和决策**。口述十分钟出提纲、分批给素材出初稿、逐轮审阅出终稿——GA 把最费时的执行环节自动化，把最需要专业判断的决策留给你。
